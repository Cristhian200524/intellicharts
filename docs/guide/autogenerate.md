# Automatic Dashboard Generation

IntelliCharts features an autogeneration engine that analyzes a dataset, infers schema data types, selects compatible charts, structures them into a responsive layout, and mounts them to the container.

---

## Vanilla JS/TS Autogeneration

A dashboard layout can be generated with a static call to `Dashboard.autogenerate`:

### 1. Simple Usage (Inference-Only)

Provide only the container element and your raw dataset (or CSV string):

```typescript
import { Dashboard } from 'intellicharts';

const container = document.getElementById('dashboard-root')!;
const myData = [
  { region: 'North', sales: 1500, date: '2026-01-01' },
  { region: 'South', sales: 2400, date: '2026-01-02' }
];

// Generates, builds, and mounts the entire dashboard automatically
const dashboard = Dashboard.autogenerate(container, myData);
```

### 2. Custom Schema & Configuration

You can override inferred data types and configure global layouts by passing the `schema` and `config` parameters:

```typescript
import { Dashboard } from 'intellicharts';

const container = document.getElementById('dashboard-root')!;

// 1. Force the engine to parse 'date' as a DataType date
const schemaOverrides = {
  date: 'date' as const,
  sales: 'float' as const
};

// 2. Set grid columns and visual theme
const layoutConfig = {
  columns: 3,
  theme: 'sketch' as const
};

const dashboard = Dashboard.autogenerate(
  container,
  myData,
  schemaOverrides,
  layoutConfig
);
```

## React Autogeneration

React developers can render an autogenerated dashboard using the declarative `<AutoDashboard />` component:

### 1. Basic Usage
```tsx
import { AutoDashboard } from 'intellicharts/react';

export default function App() {
  return <AutoDashboard data={myData} />;
}
```

### 2. Custom Properties & Themes
You can pass custom schemas, themes, and layouts directly as component props:
```tsx
import { AutoDashboard } from 'intellicharts/react';

export default function App() {
  return (
    <AutoDashboard 
      data={myData} 
      schema={{ signup_date: 'date' }} 
      theme="modern" 
      columns={3} 
      gap="25px"
    />
  );
}
```

## How the Autogeneration Engine Works

The engine processes the dataset in multiple steps to analyze and configure the dashboard:

### 1. Data Type Inference
The engine analyzes the keys and values in the provided dataset. Columns are classified into one of the following `DataType` categories:
- `'integer'`: Whole numeric numbers (formatted without decimals).
- `'float'`: Floating point numeric values (formatted with 2 decimal places in KPI cards).
- `'percentage'`: Numbers representing percentages (automatically formatted with `%`).
- `'boolean'`: True/false values.
- `'date'`: Date values (e.g., `YYYY-MM-DD`).
- `'datetime'`: Date and time values.
- `'time'`: Time-only strings (e.g., `HH:MM`).
- `'geo'`: Geographic regions, countries, states, cities, or coordinates.
- `'category'`: Low-cardinality values suited for categorizing or grouping data.
- `'string'`: Generic textual data.

#### Heuristic Rules
- Checks boolean keywords first (`true`, `false`, `yes`, `no`).
- Looks for temporal formats (`YYYY-MM-DD`, `HH:MM`, etc.).
- Identifies geographic fields by keyword checks (such as `country`, `latitude`, `postalcode`).
- Inspects numeric ranges and the presence of percentage signs `%`.
- Calculates cardinality (unique values vs total rows ratio) to distinguish low-cardinality categories from arbitrary text.

### 2. Metric Verification
If a dataset contains no numeric measure fields, the engine automatically creates a virtual measure named `__count` and sets it to `1` for all rows, allowing users to analyze records count on categorical dimensions.

### 3. Layout Planning
The engine dynamically plans a dashboard layout using a predefined capabilities dictionary:
- **KPI Cards**: Up to 3 KPI cards are added at the very top row of the dashboard for key metrics, showing aggregated totals formatted using locale-aware settings.
- **Trend Lines**: If a temporal field (`date`, `datetime`) is identified, it is combined with a key measure to build a timeline line chart spanning the full width.
- **Categorical Charts**: Category fields are mapped to a variety of chart types:
  - Low cardinality ($\le 5$ categories) uses a **pie**, **donut**, or **funnel** chart.
  - Medium cardinality ($\le 10$ categories) uses a **bar** or **radar** chart.
  - High cardinality uses a **bar** chart.
- Grid column spans are dynamically adjusted to fit within the dashboard columns constraint.
